# Internacionalização (I18N)

O sistema de internacionalização do millisecond foi implementado utilizando **next-intl**.

## 🌍 Idiomas Suportados
Atualmente suportamos dois idiomas:
- `pt-BR` (Português do Brasil) — **Idioma Padrão**
- `en` (Inglês)

## 🗂️ Estrutura de Rotas
As rotas da aplicação estão dentro da estrutura dinâmica `src/app/[locale]/`. Isso significa que todas as URLs contêm o idioma (ex: `/pt-BR/draft` ou `/en/simulate`). O redirecionamento na raiz `/` é automático para o idioma padrão caso não haja cookies prévios, feito pelo middleware `src/middleware.ts`.

## ✍️ Como Traduzir Narrativas (Narrator)
O motor de simulação de domínio em si não "fala" um idioma. Em vez de retornar strings hardcoded nas funções (`explainRace`, etc), ele retorna um objeto do tipo `NarrativeEvent`:
```typescript
type NarrativeEvent = {
  key: string;
  params?: Record<string, string | number>;
  severity?: "low" | "medium" | "high" | "critical";
};
```
### Na Camada de UI
A UI intercepta a `key` retornada (ex: `Narrator.race.neutral`) e utiliza o hook `useTranslations` do next-intl para encontrar o template textual correspondente nos JSONs de tradução (`src/messages/*.json`), injetando os `params` conforme necessário.

## 🏗️ Estratégia de Organização (Namespaces)
Atualmente, para simplificar o MVP e evitar overhead de dezenas de arquivos, mantemos **um único arquivo JSON por idioma** (ex: `en.json`, `pt-BR.json`). 
Entretanto, o conteúdo dentro desse arquivo é agrupado em "namespaces" (chaves de nível superior como `"Landing"`, `"Draft"`, `"Simulate"`, `"Narrator"`, `"Errors"`).
Quando os dicionários crescerem demais, a recomendação arquitetural será quebrar esses namespaces em arquivos isolados (ex: `messages/en/draft.json`) e configurar o plugin de i18n para carregamento por página.

## 🚫 Lidar com Nomes Próprios
Pilotos, circuitos, equipes e carros não são traduzidos em nenhuma parte do aplicativo, devendo respeitar seus nomes originais do dataset. O JSON de tradução trabalha com a interpolação: `{driverName}`, `{circuitName}`, etc.

## ➕ Como Adicionar um Novo Idioma
1. Crie um novo arquivo JSON em `src/messages/` (ex: `es.json`) com as mesmas chaves de `en.json`.
2. Adicione a chave do locale (`'es'`) no array `locales` em `src/i18n/routing.ts`.
3. Adicione a chave no array de matchers do `src/middleware.ts` na string RegExp (`/(pt-BR|en|es)/:path*`).
4. Adicione um novo botão no componente `src/components/ui/LanguageSwitcher.tsx`.

## 🧪 Testando as Chaves
Sempre que adicionar uma nova chave, lembre-se de rodar o frontend e verificar a renderização; caso uma chave não seja encontrada, o next-intl mostrará a própria chave na tela como fallback de erro. Testes de unidade em Node (como `narrator.test.ts`) devem validar se os eventos corretos estão sendo emitidos, e não o texto final em si.
